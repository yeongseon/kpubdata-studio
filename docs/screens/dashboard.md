# 화면 설계서 — 대시보드 (Dashboard)

## 화면 개요

Studio 진입 시 처음 만나는 메인 화면입니다. 전체 빌드의 상태 요약, 최근 빌드 목록, 주요 작업으로 빠르게 이동하는 진입점을 제공합니다.

## 라우트 및 진입·이탈

| 항목 | 내용 |
| :--- | :--- |
| 라우트 | `/` |
| 컴포넌트 | `src/pages/HomePage.tsx` |
| 진입점 | 앱 최초 진입, 사이드바 "대시보드" |
| 이탈점 | 최근 빌드 "보기" → [빌드 상세](build-detail.md), 빠른 시작 카드 → [새 빌드](new-build.md)·[빌드 목록](builds.md) |

## 주요 UI 구성요소

| 구성요소 | 설명 |
| :--- | :--- |
| 상태 요약 카드(4) | 대기중 · 실행중 · 성공 · 실패 건수(`listBuilds()` 집계) |
| 최근 빌드 리스트 | 최대 5건, 각 항목에 `StatusBadge` + "보기" 링크 |
| 빈 상태(EmptyState) | 빌드가 없을 때 안내 및 새 빌드 유도 |
| 빠른 시작 카드(3) | 주요 작업으로 이동하는 바로가기 |

## 상태 및 상호작용

- **로드 완료**: 요약 카드와 최근 빌드 목록 표시.
- **빈 상태**: 빌드가 하나도 없으면 `EmptyState`로 대체하고 새 빌드 만들기를 유도.
- 각 최근 빌드의 "보기" 클릭 시 해당 빌드 상세로 이동.

## 데이터 소스

- `listBuilds()` (MOCK 모드에서는 데모 시드 데이터 기반). 상태 카운트와 최근 목록 모두 이 결과에서 파생됩니다.

## 접근성

- 상태는 색상 배지와 텍스트 라벨을 함께 제공합니다.
- 카드/링크는 키보드 포커스 이동이 가능합니다.
